---
title: Type Definitions
---

import { Steps } from '@astrojs/starlight/components'

It is possible, but not required, to generate TypeScript type definitions from Go structs with `types.Generate[T]()`, where `T` is the type you wish to generate.


<Steps>
1. Define your Go types.
    ```go
    //lib/routes/handlers/welcome/types.go
    package welcome

    type Props struct {
        Message string `json:"message"`
        Error   string `json:"error"`
    }
    ```

    :::note
    Struct `json` tags are optional.
    :::
1. Call `types.Generate[T]()`.
    ```go
    //lib/routes/handlers/welcome/types.go
    package welcome

    func init() {
        _ = types.Generate[Props]()
    }

    type Props struct {
        Message string `json:"message"`
        Error   string `json:"error"`
    }
    ```
1. Start dev mode.
    ```sh
    frizzante --dev
    ```

    This will generate your type definitions in `.gen/types`.

    ```ts
    //.gen/types/gen/main/lib/routes/handlers/welcome/Props.d.ts
    export type Props = {
        message: string
        error: string
    }
    ```
    :::note
    Make sure that your Air configuration uses a `types` tag.
    ```toml
    //.air.toml
    [build]
      args_bin = []
      bin = "./.gen/tmp/main.exe"
      cmd = "go build -tags dev,trace,types -o ./.gen/tmp/main.exe ."
    ```
    Without a `types` tag, `types.Generate[T]()` is a noop.
    ```go
    //lib/core/types/generate_noop.go
    //go:build !types

    package types

    func Generate[T any]() error { return nil }
    ```
    :::
</Steps>
