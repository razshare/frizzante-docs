Views are **svelte components** exported from 
`lib/components/views/client.ts` 
and 
`lib/components/views/server.ts`.

<FileTree>
    - main.go
    - lib
        - components
            - views
                - client.ts
                - server.ts
                - Hello.svelte
                - Todos.svelte
                - ...
</FileTree>

<Tabs>
    <TabItem icon='seti:typescript' label='server.ts'>
    This file exports views that can be rendered on the server using 
    `RenderModeServer` or `RenderModeFull`.
    ```ts
    import Welcome from './Welcome.svelte'
    import Todos from './Todos.svelte'
    import type {Component} from "svelte";

    export const views: Record<string, Component> = {
        "Welcome": Welcome,
        "Todos": Todos,
    }
    ```
    </TabItem>
    <TabItem icon='seti:typescript' label='client.ts'>
    This file exports views that can be rendered on the client using `RenderModeClient` or `RenderModeFull`.
    ```ts
    export const views: Record<string, Promise<unknown>> = {
        "Welcome": import('./Welcome.svelte'),
        "Todos": import('./Todos.svelte'),
    }
    ```
    
    These views are being imported inline in order to split them 
    in different bundles, however you can simply create fake promises
    in order to get just 1 single bundle and eliminate network latency
    when transitioning between views.
    
    ```ts
    import Welcome from './Welcome.svelte'
    import Todos from './Todos.svelte'
    import type {Component} from "svelte";

    export const views: Record<string, Promise<unknown>> = {
        "Welcome": Promise.resolve(Welcome),
        "Todos": Promise.resolve(Todos),
    }
    ```
    </TabItem>
</Tabs>

## Load views

Once you've exported your views, you must load them on the server.

Add `WithPublicRoot()`, `WithViewIndex()` and `WithViewServer()` to the 
call chain in your server definition.

```go
//main.go
package main

import (
	"embed"
	"github.com/razshare/frizzante/frz"
)

//go:embed dist/*
var dist embed.FS

func main() {
	frz.NewServer().
		WithDist(dist).
		WithPublicRoot("dist/client").
		WithViewIndex("dist/client/index.html").
		WithViewServer("dist/server/server.js").
		Start()
}
```

## Send Views

You can send views with `res.SendView()`.

```go
//lib/routes/handler.go
package lib

import "github.com/razshare/frizzante/frz"

func GetHello(req *frz.Request, res *frz.Response) {
    // Sends file "lib/components/views/Hello.svelte"
    res.SendView(frz.View{Name: "Hello", RenderMode: frz.RenderModeFull}) 
}
```

The **Name** of the view will be used to lookup the view component.

Depending on the **RenderMode**, your view should be exported by either **server.ts** or **client.ts**.

| RenderModes |  |
|------------|--------|
| RenderModeServer | view component should to be exported by **server.ts**. |
| RenderModeClient | view component should to be exported by **client.ts**. |
| RenderModeFull | view component should to be exported by both **server.ts** and **client.ts**. |

:::note
The default render mode is `RenderModeFull`.
:::

## Data

Optionally, you can send data along with the views by adding a `Data` property to your `View`.

```go
//lib/routes/handler.go
package lib

import "github.com/razshare/frizzante/frz"

func GetHello(req *frz.Request, res *frz.Response) {
    res.SendView(frz.View{ 
		Name: "Hello",
		Data: map[string]string{
			"name": "world",
		}
	}) 
}
```

Then use [getContext("server")](https://svelte.dev/docs/svelte/context) to retrieve your data.

```svelte
//lib/components/views/Hello.svelte
<script lang="ts">
    import type { ServerContext } from "$frizzante/types.ts"
    const server = getContext("server") as ServerContext<{ name: string }>
</script>

<h1>Hello {server.data.name}</h1>
```

## Render Modes

You can choose how to render views with the `RenderMode` property.

<Tabs>
    <TabItem label='Server'>
    
        ```go
        //lib/routes/handler.go
        package lib

        import "github.com/razshare/frizzante/frz"

        func GetHello(req *frz.Request, res *frz.Response) {
            res.SendView(frz.View{
                Name: "Hello",
                RenderMode: frz.RenderModeServer, // Renders on server.
            })
        }
        ```
        :::tip
        While using `RenderModeServer` your application won't serve a JavaScript bundle,
        but you can still use the `<svelte:head>` special
        tag in order to load scripts dynamically.
        ```svelte
        <svelte:head>
            <script type="text/javascript" src="https://some.cdn/some.js" />
        </svelte:head>
        ```
        :::
