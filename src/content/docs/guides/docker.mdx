A Docker solution is available for both development and production deployment.

#### Get Started

<Steps>
1. Create a new container.

    ```sh
    git clone git@github.com:cmjoseph07/frizzante-docker.git
    cd frizzante-docker
    docker compose build --pull --no-cache
    ```

1. Start the container.

    ```sh
    docker compose up
    ```
    :::note
    First launch might take a minute or two because the 
    project needs to configure itself installing [Air](https://github.com/air-verse/air), [Bun](https://bun.sh/) and code dependencies.
    :::
    :::tip
    You can send the container in the background with
    ```sh
    docker compose up -d
    ```
    and you can stop it with
    ```sh
    docker compose down --remove-orphans
    ```
    :::

1. Attach to this container with your IDE or directly with a shell.

    ```sh
    docker exec -it frizzante-start sh
    ```

1. Configure project.

    ```sh
    frizzante --configure
    ```
1. Start development.

    ```sh
    frizzante --dev
    ```

1. Build.

    ```sh
    frizzante --build
    ```

    This will create a `.gen/bin/app` standalone executable.

    <FileTree>
       - .gen
          - bin
             - app
    </FileTree>
</Steps>

#### Production with Docker

You have different options for deploying to the production environment using docker.

1. **Build and run production binary inside container**

    ```sh
    # Inside the container
    frizzante --build
    ./.gen/bin/app
    ```

1. **Build production Docker image**

    ```sh
    docker build --target frizzante_prod -t my-app:prod .
    docker run -p 8080:8080 my-app:prod
    ```

1. **Use Docker Compose for production**

    ```sh
    docker compose -f compose.yaml -f compose.prod.yaml up -d --build
    ```

:::note
For more details see [Frizzante Docker](https://github.com/cmjoseph07/frizzante-docker).
:::
